# Обучение по слоям

## Идея

Обучение строится сверху вниз. Сначала модель учится выбирать крупную область смысла: предмет, действие, человек, место, эмоция, язык, число, природа, диалог, мышление, восприятие или тело. Это слой `0`, карта “материков”.

После стабилизации верхнего слоя можно добавлять уточняющий слой: конкретные страны, города и поселки внутри выбранной области. Технически это будет следующий элемент `strength_vector`, например `[3, 8]`.

## Требования

- Python 3.11+.
- PyTorch нужен только для обучения генеративного Torch-декодера. Без PyTorch обучение феромонов, память ответов, `semantic_vector` и fallback-интерпретация работают.

## Этап 1: верхний слой

Запустить курируемый curriculum:

```powershell
python3.11 -m semantic_ants learn data/top_layer_curriculum.jsonl --epochs 5 --strength-vector 3 --no-cache-refresh
```

Если `python3.11` в системе называется `python`, можно использовать `python -m semantic_ants`.

Формат строки JSONL:

```json
{
  "stimulus": "яблоко",
  "lang": "ru",
  "strength_vector": [3],
  "layer_targets": {"0": ["/m/top/object"]},
  "target_concepts": ["/m/top/object"],
  "concept_labels": {"/m/top/object": "предмет"},
  "positive_edges": [["/c/ru/яблоко", "InTopDomain", "/m/top/object"]],
  "accepted_answer": "Яблоко относится к области предметов."
}
```

`layer_targets` задает цель именно для слоя. `concept_labels` обучает человекочитаемое имя концепта, поэтому интерпретатор не держит список верхних лейблов в коде. `target_concepts` оставлен для совместимости с существующим обучением и оценкой.

`accepted_answer` нужен как обучающий сигнал, но в checkpoint сохраняется не дословная строка, а набор концептов ответа и его язык. Это позволяет генератору собирать несколько вариантов ответа и удерживать язык вопроса.

## Этап 2: проверка маршрутов

```powershell
python3.11 -m semantic_ants analyze "яблоко упало" --lang ru --strength-vector 3 --trace --json --no-cache-refresh
```

В JSON нужно проверить:

- `semantic_vector.strength_vector` равен `[3]`;
- `semantic_vector.top_domain` указывает на сильный верхний домен;
- `signal_trace` содержит `layer`, `distance`, `edge_type`, `remaining_strength`;
- в `signal_trace` для верхнего режима нет шагов глубже слоя `0`.

## Этап 3: интерпретация вектора

Готовый вектор можно декодировать отдельно:

```powershell
python3.11 -m semantic_ants interpret-vector vector.json --no-cache-refresh
```

Если Torch-модель есть, декодер использует ее. Если модели или PyTorch нет, ответ строится по правилу: главная верхняя область плюс сильнейшие активированные понятия.

## Этап 4: следующий слой

Когда верхний слой стабилен, можно переходить к вектору `[3, 8]`:

- `3` шага на карте материков;
- `8` шагов на уточняющем слое.

Для этого нужно добавлять обучающие примеры, где `layer_targets` содержит цели для слоя `1`, а графовые ребра имеют `layer=1`. Старые ConceptNet и seed-связи уже считаются слоем `1` по умолчанию.
