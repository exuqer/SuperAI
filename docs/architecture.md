# Архитектура

## Поток данных

1. `core.normalization` очищает текст, определяет `ru/en` и строит ConceptNet URI.
2. `providers.conceptnet` загружает связи ConceptNet и WordNet-derived ребра.
3. `knowledge.base` автозагружает алфавиты, частые слова, базовые смыслы и диалоговые реакции.
4. `core.graph` собирает локальный граф для текущей фразы.
5. `ants.colony` запускает муравьев по графу с учетом феромонов checkpoint.
6. `generation.interpreter` ранжирует активированные концепты и строит ответ.
7. `learning` усиливает или ослабляет маршруты по JSONL и feedback.

## Разделение знаний

Внешний граф считается базой фактов. Он не переписывается. Все обучение хранится в overlay:

- `pheromones`: веса доверия к маршрутам;
- `custom_edges`: новые пользовательские связи;
- `suppressed_concepts`: штрафы для ложных смыслов;
- `response_memory`: запомненные ответы для наборов концептов.
- `metadata`: алфавиты, частые слова, версии встроенных seed-данных.

Такое разделение позволяет позже заменить ConceptNet-провайдер, генератор или роутер без потери обученного слоя.
