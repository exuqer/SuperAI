# Операции runtime

- Повторить после сбоя: перезапустить процесс; только `running` work items
  вернутся в `queued`.
- Найти неразрешимую команду: `GET /api/v1/system/dead-letters`.
- Проверить трассу: `GET /api/v1/traces/{trace_id}`.
- Не исправлять состояние через SQLite вручную: повторить команду с тем же
  idempotency key либо создать новую версионированную команду.
