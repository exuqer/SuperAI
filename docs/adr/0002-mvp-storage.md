# ADR 0002: SQLite metadata + content-addressed files

Статус: принято.

Канонические метаданные, очередь, трассы и снимки хранятся в SQLite. Байты
артефактов сохраняются в файловом content-addressed store по SHA-256.
Логический `artifact_id` не равен `content_hash`: одинаковые байты могут иметь
разные владельческие записи, а одна логическая запись может иметь версию.

Запись сначала попадает во временный файл, fsync-ится и атомарно публикуется;
затем в транзакции фиксируются метаданные. Чтение сверяет hash. Сборка мусора
использует mark-and-sweep от корней, а не только счётчики ссылок.
