# Декодер

`/api/decode` и страница `/decode` делают обратный диагностический проход:

`tokens/text -> роли -> формы слов -> нормальная фраза`

Этот слой нужен для проверки того, как из токенов собирается осмысленная фраза. Он не пишет в checkpoint, не трогает чат и не меняет `/api/analyze`.

## Вход

Приоритет у `tokens`. Если `tokens` пустой, декодер берет токены из `text`.

Пример запроса:

```json
{
  "text": "кот есть рыба мясо",
  "tokens": ["кот", "есть", "рыба", "мясо"],
  "lang": "auto",
  "session_id": "optional",
  "turn_id": "optional"
}
```

## Правила

- Входные токены можно передавать не в порядке готового предложения.
- Для русского декодер сначала ищет глагол по морфологии.
- Субъект выбирается среди существительных и местоимений.
- Для переходных глаголов одушевленный токен имеет приоритет как возможный субъект.
- Простые временные слова вроде `осень`, `зима`, `день` могут стать обстоятельством.
- Глаголы-связки изменения состояния вроде `становиться` собираются как `обстоятельство + субъект + глагол + признак`.
- Русский субъект инфлектится в `nomn sing`.
- Русский глагол инфлектится в `3per sing pres`.
- Русские объекты инфлектятся в `accs sing`.
- Русский признак после `становиться` инфлектится в творительный падеж с согласованием по субъекту.
- Английский глагол переводится в `3rd-person singular`.
- Объекты соединяются через `и` или `and`.

Примеры:

```text
кот, есть, рыба, мясо -> кот ест рыбу и мясо
шкаф, собирать, рабочий -> рабочий собирает шкаф
осень, лист, становиться, желтый -> осенью лист становится жёлтым
```

## Выход

Ответ содержит:

- исходный текст и токены;
- выбранный язык;
- итоговую `sentence`;
- `pattern`;
- список токенов с ролями и поверхностными формами;
- краткий `summary`.

Пример:

```json
{
  "sentence": "кот ест рыбу и мясо",
  "pattern": "svo",
  "summary": {
    "total_tokens": 4,
    "used_tokens": 4,
    "objects": 2,
    "fallbacks": 0
  }
}
```

## Ограничения

- Свободный порядок слов поддерживается только эвристически.
- Несколько субъектов не поддерживаются.
- Согласование по числу и падежу для сложных случаев будет следующим этапом.
