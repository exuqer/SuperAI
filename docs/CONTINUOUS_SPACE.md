# Непрерывное пространство понятий

## Короткий ответ

Да, визуализация представляет собой непрерывное двумерное поле с плавным
изменением масштаба. Но это не набор дискретных экранов, расположенных друг
над другом, и не `z-index` в смысле CSS.

У пространства есть две разные координаты:

- `x`, `y` — положение объекта в общем двумерном поле;
- `zoom` — масштаб камеры, то есть степень приближения к выбранной области.

Из `zoom` вычисляется непрерывная семантическая глубина. Она управляет тем,
какие представления объектов становятся видимыми, но сами данные не
перескакивают на другой физический слой и не перемещаются в отдельный экран.

Схема работы:

```text
единое поле x/y
        |
        v
камера: position + zoom
        |
        v
semanticDepth = clamp(log2(zoom) + 1, 0, 3)
        |
        v
плавное смешивание представлений:
сцены -> понятия -> словоформы -> символы
```

## Что является непрерывным

### Поле координат

Все объекты получают координаты в двумерном пространстве. Камера показывает
прямоугольную область этого поля. При увеличении масштаба центр камеры остаётся
тем же, а видимая область становится меньше; при уменьшении масштаба видимая
область расширяется.

Для точки мирового пространства `(worldX, worldY)` её положение на экране
вычисляется так:

```text
screenX = (worldX - cameraX) * zoom + viewportWidth / 2
screenY = (worldY - cameraY) * zoom + viewportHeight / 2
```

Обратное преобразование используется для наведения мыши и панорамирования:

```text
worldX = (screenX - viewportWidth / 2) / zoom + cameraX
worldY = (screenY - viewportHeight / 2) / zoom + cameraY
```

Поэтому масштабирование относительно курсора сохраняет под курсором одну и ту
же мировую точку. Это обычная камера над бесшовным 2D-полем, а не переход
между заранее отрисованными страницами.

### Масштаб камеры

В `SpaceVisualization` масштаб ограничен диапазоном `0.22..64`. Это технический
предел интерфейса, а не границы математического пространства. Значения внутри
этого диапазона непрерывны: колесо мыши умножает текущий масштаб на плавный
коэффициент, а drag изменяет положение камеры.

`zoom = 1` — исходный масштаб. Например:

| `zoom` | Видимый смысл |
| ---: | --- |
| `0.22` | максимально широкий обзор |
| `0.5` | обзор крупного контекста |
| `1` | исходный обзор поля |
| `2` | приближение примерно в два раза |
| `4` | глубокий просмотр |
| `64` | технический максимум приближения |

## Что является уровнями

Уровни — это не CSS-слои с разными `z-index`. Это логические представления
разной гранулярности, которые рендерятся поверх одного поля и плавно меняют
прозрачность.

В текущем интерфейсе используются четыре представления:

| Представление | `layer` | Смысл | Ориентировочная глубина |
| --- | --- | --- | ---: |
| Сцены | `scene` | целое предложение или крупный контекст | `0` |
| Понятия | `concept` | семантические области и их соседи | `1` |
| Словоформы | `word_form` | конкретные слова внутри сцены | `2` |
| Символы | `character` | символы, из которых состоят слова | `3` |

Глубина вычисляется формулой:

```ts
semanticDepth = clamp(Math.log2(zoom) + 1, 0, 3)
```

Из неё следуют приблизительные опорные точки:

```text
semanticDepth 0 -> zoom 0.5  -> сцены
semanticDepth 1 -> zoom 1    -> понятия
semanticDepth 2 -> zoom 2    -> словоформы
semanticDepth 3 -> zoom 4    -> символы
```

Это логарифмическая шкала. Приближение от `1` до `2` повышает глубину на один
семантический шаг, как и приближение от `2` до `4`. Поэтому равные изменения
`zoom` не должны использоваться как равные изменения смыслового уровня.

## Почему одновременно видны несколько уровней

Каждый уровень имеет диапазон появления и исчезновения. Между границами
используется `smoothstep`, поэтому объект не включается в один момент и не
исчезает в другой момент скачком.

Основные диапазоны по `semanticDepth`:

| Представление | Появление | Исчезновение |
| --- | ---: | ---: |
| Сцены | `0` | `0.72..1.42` |
| Понятия | `0.62..1.22` | `1.62..2.18` |
| Словоформы | `1.35..1.92` | `2.36..2.82` |
| Символы | `2.30..2.90` | не исчезают до глубины `3` |

Следствие: в переходных областях два соседних представления видны одновременно.
Например, при уходе от сцен к понятиям сцены ещё сохраняются как слабые ореолы,
а понятия уже проявляются. Это создаёт эффект непрерывного раскрытия структуры:
родительский объект не исчезает мгновенно, а становится контекстом для более
детального представления.

## Как раскрываются дочерние объекты

Нижний уровень не просто появляется в случайном месте.

Для словоформ используется интерполяция от позиции родительской сцены к их
собственной позиции:

```text
wordPosition = scenePosition + (wordTarget - scenePosition) * reveal
```

где `reveal` плавно меняется от `0` до `1` в диапазоне глубины `0.92..1.78`.

Для символов применяется такой же принцип относительно словоформы:

```text
characterPosition = wordPosition +
                    (characterTarget - wordPosition) * reveal
```

`reveal` для символов меняется от `0` до `1` в диапазоне `2.02..2.78`.

Таким образом, дочерние элементы визуально расходятся из родительского центра
по мере приближения. Это не означает, что родитель физически содержит дочерние
объекты в третьем измерении; это переход между двумя представлениями одной
иерархии.

## Что означает `z-index`

В этой модели `z-index` не кодирует смысловую глубину.

В SVG порядок групп определяет порядок отрисовки, а прозрачность определяет,
насколько виден каждый уровень. В CSS `z-index: 5` у debug-панели и подсказки
нужен только для того, чтобы UI-оверлеи были поверх canvas/SVG. Он не означает,
что отладочная панель находится на пятом уровне понятий.

Иными словами:

```text
z-index      -> технический порядок наложения интерфейсных элементов
semanticDepth -> степень раскрытия данных
zoom          -> масштаб камеры
x/y           -> положение в общем поле
```

Смешивать эти понятия нельзя.

## Семантическое и структурное приближение

В проекте есть два связанных, но разных механизма.

### Визуальный непрерывный zoom

`SpaceVisualization.vue` получает иерархию через
`GET /api/field/hierarchy?max_depth=3`, затем в одном SVG-поле показывает сцены,
семантические оверлеи, слова и символы. При изменении `zoom` меняются глубина,
прозрачность и позиции представлений.

Это основной механизм непрерывного обзора.

### Серверная навигация по пространствам

`ZoomService` хранит путь текущей сессии (`ZoomPath`) и умеет открывать
отдельные локальные пространства:

- `zoom_in_structural` — структурное пространство дочерних компонентов,
  обычно переход к более низкому `layer_id`;
- `zoom_in_semantic` — семантическое пространство соседей на том же
  `layer_id`;
- `zoom_out` — возврат по breadcrumb к родительскому пространству.

Эти пространства являются логическими контейнерами данных и могут быть
созданы лениво. Их `mode` (`structural` или `semantic`) и `layer_id` — не
аналог CSS `z-index` и не дискретные координаты камеры. Серверная навигация
определяет состав данных локального пространства, а клиентский `zoom`
определяет, как эти данные визуально раскрываются.

## Что происходит при приближении

```text
1. Пользователь вращает колесо или нажимает +/−.
2. Вычисляется новый zoom и, если нужно, новая позиция камеры,
   чтобы точка под курсором осталась неподвижной.
3. Из zoom вычисляется semanticDepth.
4. Для каждого уровня вычисляется opacity через плавные функции.
5. Позиции слов и символов интерполируются от позиций родителей
   к собственным координатам.
6. Выбирается подпись текущего масштаба:
   «Предложения», «Понятия», «Слова» или «Буквы».
7. Компонент сообщает наружу `{ x, y, zoom }` через событие `camera-change`.
```

## Что происходит при обучении

Обучение не создаёт отдельный `z-index` для каждого нового понятия.

Поле хранит глобальные облака. У облака есть, среди прочего, `layer_id`, масса,
плотность, радиус, активация и каноническое имя. Позиция и масса влияют на
геометрию и внешний вид. Один и тот же глобальный cloud может иметь локальные
размещения (`CloudPlacement`) в разных пространствах.

Структурная связь описывается через `StructuralComponent`, а локальное
размещение содержит собственные `x`, `y`, `z`, радиус и состояние активации.
Поле визуализации сейчас двумерное (`coordinate_dimensions = 2`), поэтому
`z` в модели размещения не является текущим управляющим параметром камеры.

Вероятность совместной активации также не выводится из `z-index`. Она связана
с геометрией поля, расстоянием, массой, активацией и статистикой совместной
активации.

## Итоговая формулировка

Корректнее всего описывать систему так:

> Это непрерывное двумерное поле с камерой, которая плавно меняет масштаб.
> Семантическая глубина вычисляется из масштаба логарифмически и управляет
> плавным смешиванием нескольких представлений одной иерархии: от сцен к
> понятиям, словам и символам. Дискретные структурные и семантические
> пространства существуют на сервере как контейнеры навигации, но не являются
> `z-index` и не превращают визуализацию в набор независимых экранов.
