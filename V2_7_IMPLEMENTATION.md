# SuperAI V2.7 — статус реализации

## Выполнено

- свежая SQLite-схема без ролевых полей, ограничений и индексов;
- несовместимая прежняя БД удаляется перед созданием V2.7, без legacy backfill;
- top-K морфологические гипотезы с поздней фиксацией;
- полные квалифицированные упоминания и обязательные компоненты;
- события с неназванными участниками и разреженными сигнатурами;
- мягкие локальные слоты, slot sets, merge/split, ослабление и прототипы;
- анонимные контекстные кластеры сущностей с возможностью split;
- обучаемые структурные конструкции;
- QueryGraph и семь универсальных видов gap;
- контекстная фиксация вопросительных омонимов без отображения в роли;
- строгий графовый допуск и CandidateBinding;
- активные/пассивные варианты и свободный порядок слов;
- `NODE_COMPONENT` для вопросительного определения;
- `RELATION_VALUE` и `QUANTITY_VALUE` для наблюдаемых связей и чисел;
- продолжения с новым gap и повтор gap с исключениями;
- каноническая дедупликация и число независимых источников;
- частичный успех короткой/полной генерации и обратная проверка;
- безопасные training episodes без самоусиления фактов;
- staging, commit, reprocess и retraction;
- изолированные batch preview/commit/rollback с проверяемым жизненным циклом;
- полный trace и версии всех компонентов;
- новый HTTP API и диагностическая регрессия V2.7.

## Результат аудита

Активный маршрут:

```text
server.server
→ modules.training / modules.hive
→ graph_service
→ event_graph / query_graph
→ graph_learning / graph_models
→ graph_schema
```

В этом маршруте отсутствуют `requested_role`, `fixed_roles`, `semantic_role`,
ролевые индексы и ручная валентность. Старый интерпретатор именованных ролей
удалён из языкового маршрута; отрицание сохраняет только сигнатуру наблюдаемого
узла. Старые endpoint-роутеры и schema-backfill не подключаются.

## Измеренные ворота

- диагностическая регрессия: `31 passed`;
- конфликт обязательного компонента: отклоняется до ranking;
- перестановки и события с тремя участниками различаются совместной оценкой
  известных узлов и gap;
- форма короткого ответа восстанавливается из морфологии gap, в том числе из
  единственного пассивного источника;
- продолжение `ещё`: два значения и затем `UNRESOLVED`;
- restart диалога: QueryGraph и исключения восстанавливаются из SQLite;
- одинаковые тексты в разных диалогах получают разные сохраняемые QueryGraph ID;
- batch commit/rollback затрагивает только источники указанного batch;
- повторный stage не понижает подтверждённый источник, а ручной commit не
  превращает вопрос в факт мира;
- support прототипа равен сумме уникальных слотов и не растёт повторно;
- удаление/переименование `display_label`: ответ не меняется;
- активный антихардкод-аудит запрещённых контрактов: 0 совпадений.
- несовместимая схема пересоздаётся, legacy-таблицы не сохраняются.

## Ещё не измерено

В репозитории нет размеченных наборов на 1 000, 20 000, 500 000 и 100 000
событий. Поэтому пороги 97%, 95%, падение не более 5 п.п. и стабильность
латентности на 100 000 событий пока не объявляются достигнутыми. Контур,
индексы и тестовые hooks для этих измерений готовы, но сами корпусные прогоны
нужно выполнять отдельной benchmark-партией.
